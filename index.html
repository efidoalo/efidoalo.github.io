Web page retrieval using the linux kernel's BSD sockets implementation and the C programming language

A program that establishes an IPv4 connection using the TCP transport layer
protocol between the local machine that runs the program and an arbitrary web
server. The connection is established on port 80 of the web server, so the server
must support this, and the application layer HTTP protocol is used to send the
appropriate get request to the server to retrieve the user specified web page.
The web servers response (HTTP) message is written to the terminal in full, so
that the web page is shown in the server (HTTP) response message body in html.

Film search website built using HTML, CSS, Javascript, PHP and mySQL

Responsive website (http://findafilm.host22.com) designed for users
to search and locate films that match their criteria. The user can specify their
film criteria on one web page and submit the criteria in JSON format to the web server
using a search button. A PHP script on the web server parses the JSON film criteria
string and makes relevant calls using the API provided by the movie database (TMDB) to
find relevant films. The API provided by TMDB is freely accessible, for more
details visit their website (www.themoviedb.org). A mySQL database which holds a subset
of TMDB is also queried to find matching films, this avoids excessive TMDB API calls.
The relevant films matching the users criteria are then sent back to their machine
to be viewed.

Contacts Manager command line application built using C and PostgreSQL

A program developed for my own personal use to manage my personal contacts. A
PostgreSQL database was created called contacts that I can connect to from
the linux postgres account. A table called contact_data was created within this
database for the C program to interact with. The program reads commands from
the terminal and using simple syntax allows inserting, deleting, searching and
listing of contacts by their first and/or last names. Commands are read from
the terminal, parsed, and relevant PostgreSQL queries are executed using the
PostgreSQL C API library libpq.

Data containers capable of holding general data types. Written using C.

Multiple header files, whereby each data structures implementation is contained
within its own seperate header file. Each data structure is designed to contain
one specific data type, that may be user defined or a built in type and whose size
in bytes is given during container initialization. Each container has an initialization
function that must be called to allocated the appropriate amount of memory for the
data structure. General data types are supported within each container by use and
casting of void pointers. Currently implemented data structures include a vector,
unidirectional and bidirectional linked lists, and a binary tree thay maintains its
balance on both insertion and deletion. Each data structure implementation also
contains a function to free all associated memory.
